**[Prototype Assignment]{.underline}**

You are required to develop a web application with the requirements
given below:

**Home Page**

- Search bar to enter a query like:*"Plan a 3-day trip to northern Pakistan under 25,000 PKR"*
- On search, extract basic parameters (destination, days, budget)
  using simple string parsing or a mock NLP module.
- Show sample "Recommended Destinations" cards dynamically (data from
  JSON, mock database, or static API).
- Guest User Access:

  - One-time or guest users should be able to browse and use the app
    without registration.
  - Registration/login should be optional for saving trips or
    accessing personalized plans.
- Include real-time or mock "Travel Suggestions" such as:

  - "Top Adventure Spots under 20,000 PKR"
  - "Best Weekend Destinations Near You"

**User Registration and Profile Management**

- Create a registration/login form (email, name, password).
- Profile page should allow saving travel preferences such as:

  - Budget range
  - Travel style (Adventure, Relaxation, Family, etc.)
  - Duration

### **Destination Recommendation**

- Data sources include public APIs (e.g., TripAdvisor, Google Places)
  and a prebuilt database of destinations.
- Create a JSON file or database table named destinations with sample
  data.
- Each destination entry in the dataset must include the following
  attributes:

---

  **Attribute**   **Description**

---

  name            Name of the destination

  type            Type of travel (Adventure, Relaxation, Family, Cultural,
                  Historical, etc.)

  region          Geographical region or province

  cost            Average trip cost in PKR

  weather         Typical weather (Cold, Warm, Moderate, etc.)

  best_season     Recommended season to visit (e.g., Summer, Winter, All
                  Year)

  activities      List of activities available (e.g., hiking, sightseeing,
                  boating)

  safety_rating   A rating from 1--5 based on travel safety

  user_rating     Average user satisfaction rating from 1--5

image           File path or image URL

- Use **Machine Learning (ML)** algorithms such as **Content-Based
  Filtering** and **Collaborative Filtering** to recommend
  destinations.
- The model learns correlations between user preferences and
  destination features.
- **Example:** If a user likes "nature" and "mountains," the system
  may recommend "Hunza, Naran, or Murree."
- For users without history, the system uses a **cold-start
  heuristic** based on popular destinations in their region.
- Data can come from a static JSON dataset or a small database table.
- Display recommended destinations with **images, costs, and weather**
  info.

### **Budget Estimation & Cost Optimization**

- Calculate an approximate total cost (hotel + travel + food).
- Use dummy average rates (stored in JSON/database).
- Show a summary table:

---

  **Item**                  **Cost (PKR)**

---

  Hotel                     10,000

  Travel                    8,000

  Meals                     5,000

**Total**                 **23,000**
----------------------------

### **Admin Dashboard**

- Admin can manage:

  - Destination data, pricing, and seasonal updates.
  - User activity statistics and feedback.
- Admin can add new regions, AI models, or update recommendation
  datasets.

**Database:** MySQL / MongoDB
